CP08-stats: Leaderboard and AI Stats

GitHub Commit: Link

What I built: Added backend endpoints (/stats/leaderboard and /stats/ai) to calculate and return player and AI stats. Created a frontend modal to display leaderboard and AI win rates by difficulty and personality.

What was hard: Initially, the leaderboard logic miscounted wins/losses because it didn't account for the player's symbol (X or O). Fixed by assuming the human player is always X and the AI is O.

What I'd do differently: I would store the player's symbol (X or O) in games.json to avoid assumptions in the stats logic.

AI usage note: Used AI to help design the stats logic and frontend tables, but manually verified the calculations.

A/B testing note: Tried displaying stats in a single table first, but switched to tabs for better readability.

CP09-c1: AI vs AI Mode

GitHub Commit: Link

What I built: Added AI vs AI mode where two AIs play against each other. Users can configure difficulty (Easy/Medium/Hard) and personality (Aggressive/Defensive/Chill) for both AIs. The game runs automatically with a delay between moves for better UX.

What was hard: Ensuring the AI 2 settings (difficulty/personality) were properly enabled and used during the game. Also, handling the recursive AI moves without breaking the game state.

What I'd do differently: I would add a "Stop Game" button to let users interrupt the AI vs AI game if they want to change settings mid-game.

AI usage note: Used AI to help design the recursive logic for alternating AI turns and debugging the event listeners for AI 2 settings.

A/B testing note: Tried using `setTimeout` vs. `setInterval` for delays between AI moves, but `setTimeout` worked better for this use case.

CP10-c2: Chat with AI

GitHub Commit: Link

What I built: Added a chat feature that allows players to communicate with the AI during PvAI and AI vs AI games. The AI responds based on its selected personality (Aggressive, Defensive, or Chill) using the Groq API.

What was hard: Initially, the AI kept responding with "Sorry, I didn't understand that" because the personality parameter wasn't being passed correctly to the backend. Fixed by properly formatting the system prompts for each personality type.

What I'd do differently: I would add a "Clear Chat" button to let users reset the conversation, and maybe save chat history to localStorage so it persists between page refreshes.

AI usage note: Used AI to help design the chat UI layout and craft personality-specific system prompts for the Groq API. Also used AI to debug the endpoint when responses weren't working.

A/B testing note: Tried different prompt styles (short vs. long, strict vs. loose instructions) and found that short, personality-specific prompts worked best for getting consistent, on-brand responses.
